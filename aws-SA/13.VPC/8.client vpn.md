# 🔐 Site-to-Site VPN (Core Idea)

From your notes 

👉 Connect:

* Your **data center (on-prem)**
* To your **AWS VPC**

👉 Over:

* Public internet
* But **encrypted (IPSec)**

---

# 🧠 Architecture (Must Remember)

```text
On-Prem (VPN device)
        ↓
Customer Gateway (CGW)
        ↓
Internet (IPSec tunnels)
        ↓
Virtual Private Gateway (VGW)
        ↓
VPC
```

---

# 🔑 Key Components

| Component      | Role                    |
| -------------- | ----------------------- |
| CGW            | Your on-prem VPN device |
| VGW            | AWS side VPN gateway    |
| VPN Connection | Tunnel between them     |
| IPSec          | Encryption              |

---

* You manually configure routes

❌ Not scalable

---

## 2️⃣ Dynamic Routing (BGP) ✅

👉 Real-world preferred

---

# ⚠️ Important Networking Rules (VERY IMPORTANT)

---

## ❌ NAT Gateway Limitation

From your notes 

```text
On-prem → VPN → VPC → NAT Gateway → Internet
```

❌ NOT allowed

👉 NAT Gateway **blocks VPN-origin traffic**

---

## ✅ NAT Instance Works

```text
On-prem → VPN → NAT Instance → Internet
```

✔ Works

👉 Because you control it

---

## ✅ Reverse Works (Common setup)

```text
VPC → VPN → On-prem → Internet
```

✔ Works

👉 Use on-prem NAT

---

# 🧠 Why This Matters

👉 AWS controls NAT Gateway
👉 Doesn’t allow “edge-to-edge” routing

# ⚠️ Scaling Problem

From your notes 

👉 If you have multiple VPCs:

```text
On-prem → VPC1 (VPN)
On-prem → VPC2 (VPN)
On-prem → VPC3 (VPN)
```

❌ Gets messy

---

# 🔥 Solution: Shared Services VPC

---

## 🧩 Architecture

```text
On-prem → VPN → Shared VPC
                  ↑
         Peering with other VPCs
```

---

## 💡 Idea

* Only **1 VPN connection**
* Other VPCs connect via **peering**

---

## ⚠️ Important Detail

From your notes 

👉 Peering is NOT transitive

So:

* VPC A ❌ cannot directly reach on-prem

---

### ✅ Workaround:

* Replicate services
  OR
* Use proxy in shared VPC

---

# 🚀 Real-World Better Solution

Instead of this complex setup:

👉 Use **Transit Gateway (TGW)**

Because:

* Supports transitive routing
* Scales cleanly

---
