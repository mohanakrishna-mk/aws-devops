# 🔐 What is a VPC Endpoint Policy?

From your notes 

👉 It’s a **JSON policy attached to the VPC endpoint**
👉 Controls **who can access which AWS service via that endpoint**

---

# 🧠 Core Idea (Very Important)

Think like this:

```id="j8p1z9"
IAM → Service Policy → VPC Endpoint Policy
```

👉 Access is allowed **ONLY if all layers allow**

---

# 🔑 Example (from your notes)

From 

👉 Allow:

* Specific IAM user
* Only `SendMessage`
* Only to one SQS queue
* Only via VPC endpoint

---

# ⚠️ Important Trap

👉 VPC Endpoint Policy applies **ONLY when traffic uses endpoint**

So:

```id="f3k7vn"
EC2 → Public Internet → S3
```

✔ Endpoint policy is **ignored**

# 🔥 How to FORCE usage of VPC Endpoint

From your notes

```id="l0d2mk"

```

# 🧩 Security Layers (Real Understanding)

To access S3 from EC2 privately, ALL must allow:

### 1️⃣ Security Group

* Outbound allowed?

---

### 2️⃣ VPC Endpoint Policy

* Allows access?

---

### 3️⃣ Route Table

* Has route to endpoint?

---

### 4️⃣ DNS

* Enabled?

---

### 5️⃣ S3 Bucket Policy

* Allows access?

---

### 6️⃣ IAM Role

* EC2 role allows S3?

---

👉 If ANY fails → ❌ Access denied

---

# 🔐 Source Conditions (VERY IMPORTANT)

From your file 

---

## ✅ aws:SourceVpce

* Restrict to specific endpoint

---

## ✅ aws:SourceVpc

* Restrict to specific VPC

---

## ❌ aws:SourceIp (common mistake)

* Works ONLY for public IP
* NOT for private VPC traffic

# 🧠 Big Picture (Don’t Miss This)

👉 VPC Endpoint Policy = **network-level filter**

BUT:

* Does NOT replace IAM
* Does NOT replace bucket policy

👉 It’s an **extra layer**

# 🎯 Interview-Level Insight

✔ Answer:
