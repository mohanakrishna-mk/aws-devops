# 🚀 What is AWS Network Firewall?

From your notes 

👉 A **managed firewall for your entire VPC**

👉 Works at:

* Layer 3 → Layer 7

👉 Inspects:

* Inbound
* Outbound
* VPC-to-VPC
* VPN / Direct Connect traffic

# 🏗️ Real Architecture (VERY IMPORTANT)

---

## 🧩 Inspection VPC Pattern

```text id="c4m97l"
VPCs → Transit Gateway → Inspection VPC (Firewall) → Destination
```

---

👉 All traffic goes through:

* Transit Gateway
* Network Firewall

---

# 🔥 3 Traffic Patterns (Must Understand)

---

## 1️⃣ North-South (Internet Traffic)

```text id="u8v9t1"
App VPC → TGW → Firewall → TGW → Egress VPC → Internet
```

---

## 2️⃣ East-West (VPC to VPC)

```text id="xbl15v"
VPC A → TGW → Firewall → TGW → VPC B
```

---

## 3️⃣ Hybrid (On-Prem)

```text id="cgrsnb"
VPC → TGW → Firewall → TGW → VPN/DX → On-prem
```

---

# ⚠️ Key Insight

👉 Firewall is NOT directly in each VPC

👉 It sits in a:

* **central inspection VPC**

---

# 🧠 Why Transit Gateway is Required

Because:

* You need **central routing control**
* You want all traffic to pass through firewall
