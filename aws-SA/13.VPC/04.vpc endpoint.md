# 🚀 What is a VPC Endpoint (simple)

From your notes 

👉 It lets your EC2 (or resources) talk to AWS services
👉 **WITHOUT using internet / NAT / IGW**

---

# 🧠 Core Idea

Instead of:

```id="qv0h5n"
EC2 → NAT → Internet → S3
```

You do:

```id="zjzqk8"
EC2 → VPC Endpoint → S3 (private AWS network)
```

👉 More secure
👉 Cheaper (no NAT data cost)
👉 Faster

---

# 🔑 Two Types of VPC Endpoints

---

# 1️⃣ Gateway Endpoint (VERY COMMON)

From your file 

### Works ONLY for:

* S3
* DynamoDB

---

### How it works:

* Added at **VPC level**
* Uses **route tables**

Example:

```id="g4d9x7"
Destination: S3
Target: VPC Endpoint
```

👉 Traffic automatically goes private

---

### 💡 Key Points:

* No ENI created
* No security group
* Free (big advantage)
* Must update route tables

---

### ⚠️ Limitation:

👉 Cannot be used via:

* VPC Peering
* Transit Gateway
* VPN / Direct Connect

👉 Only works **inside same VPC**

---

# 2️⃣ Interface Endpoint (PrivateLink)

From your notes 

### Works for:

* Almost ALL AWS services (CloudWatch, SNS, SQS, etc.)

---

### How it works:

* Creates **ENI (private IP)** inside subnet
* Uses **security groups**

---

### 💡 Key Points:

* Supports private DNS
* Works across:

  * VPN
  * Direct Connect
* Chargeable

---

# 🔐 Private DNS (Important)

👉 If enabled:

* You can still use normal AWS URL
* It resolves to **private IP (endpoint ENI)**

Example:

```id="r1b6d9"
s3.amazonaws.com → private IP
```

---

👉 If NOT enabled:

* You must use special endpoint URL

---

# ⚠️ Common Issues (Real Debugging)

From your file 

If endpoint not working, check:

* DNS enabled?

  * Enable DNS support
  * Enable DNS hostname
* Route table (gateway endpoint)
* Security group (interface endpoint)

# ⚔️ Gateway vs Interface (Important Table)

| Feature           | Gateway Endpoint | Interface Endpoint |
| ----------------- | ---------------- | ------------------ |
| Services          | S3, DynamoDB     | Almost all         |
| Cost              | Free             | Paid               |
| Uses route table  | ✅                | ❌                  |
| Uses ENI          | ❌                | ✅                  |
| Security groups   | ❌                | ✅                  |
| Works via TGW/VPN | ❌                | ✅                  |

# 🧠 Hidden Interview Traps

👉 “Private subnet EC2 cannot access S3”

✔ Solution:

* Add **Gateway Endpoint**

NOT:

* NAT Gateway (costly mistake)

---

👉 “Access AWS service privately from on-prem”

✔ Solution:

* Interface Endpoint

NOT:

* Gateway Endpoint

---

# 🎯 Simple Memory Trick

* **Gateway = S3/DynamoDB shortcut**
* **Interface = private ENI access to AWS services**

---

# ⚠️ Important Restrictions (must remember)

From your notes

* Not accessible via:

  * Peering
  * Transit Gateway
  * VPN

👉 Interface endpoint:

* Can be accessed across networks
