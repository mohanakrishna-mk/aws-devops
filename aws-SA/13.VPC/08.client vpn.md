You’ve now hit the **last piece of AWS networking**: **AWS Client VPN** — this is about *user access*, not system-to-system networking.

---

# 🚀 What is AWS Client VPN?

👉 It lets **your laptop connect securely to AWS**

👉 Uses:

* OpenVPN protocol
* Public internet
* Encrypted tunnel

---

# 🧠 Core Idea

```text
Your Laptop → Internet → Client VPN → VPC → Resources
```

👉 After connecting:

* You behave like you're **inside the VPC**

* Authentication:

  * Certificates

---

# ⚔️ Client VPN vs Site-to-Site VPN

| Feature      | Client VPN             | Site-to-Site VPN       |
| ------------ | ---------------------- | ---------------------- |
| Who connects | Users                  | Networks               |
| Use case     | Developer/admin access | Data center connection |
| Device       | Laptop                 | Router                 |

# ⚠️ Common Mistakes

👉 “VPN connected but can’t access EC2”

Check:

* Route table
* Security group
* Authorization rules

---

👉 “Can’t access internet”

Check:

* IGW or NAT setup
* VPN route (0.0.0.0/0)

---

# 🧠 Memory Trick

* Client VPN = **user tunnel**
* Site-to-Site = **network tunnel**
* TGW = **network hub**
