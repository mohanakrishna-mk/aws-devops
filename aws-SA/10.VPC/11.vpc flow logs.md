# 📊 What are VPC Flow Logs?

From your notes 

👉 Capture **network traffic metadata** inside your VPC

👉 Can be enabled at:

* VPC level
* Subnet level
* ENI (instance level)

---

# 🧠 Core Idea

Think:

```text
"Who is talking to whom, on which port, and was it allowed?"
```

---

# 📦 Where Flow Logs Go

From your notes 

👉 You can send logs to:

* CloudWatch Logs
* S3
* Kinesis Data Firehose

---

# 🔍 What Data You Get

Typical fields (don’t memorize, just understand):

* Source IP
* Destination IP
* Source Port
* Destination Port
* Protocol
* Bytes
* Action → `ACCEPT` / `REJECT`

---

# 🔥 Most Important Field

👉 `action`

| Value  | Meaning |
| ------ | ------- |
| ACCEPT | Allowed |
| REJECT | Blocked |

# 🔐 SG vs NACL Debug Trick (VERY IMPORTANT)

From your notes 

---

## 🧩 Case 1

```text
Inbound → REJECT
```

👉 Could be:

* Security Group ❌
* NACL ❌

---

## 🧩 Case 2 (Important)

```text
Inbound → ACCEPT
Outbound → REJECT
```

👉 ONLY NACL problem

Why?

* Security Groups are **stateful**
* Return traffic should be allowed

---

## 🧩 Case 3

```text
Outbound → ACCEPT
Inbound → REJECT
```

👉 Again → NACL issue

---

# ⚠️ Key Concept

* **Security Group = Stateful**
* **NACL = Stateless**

👉 Flow logs help you identify which one is blocking

# 🚨 Advanced Case: NAT Gateway Confusion

From your notes 

---

## ❓ Problem:

You see:

```text
Public IP → NAT Gateway → ACCEPT
```

👉 Looks like inbound traffic allowed

---

## ❗ Reality:

* NAT Gateway **does NOT accept inbound traffic**

---

## 💡 What’s happening:

1. Traffic reaches ENI
2. Allowed by:

   * Security Group
   * NACL
3. BUT:

   * NAT Gateway drops it internally

---

👉 Flow logs show ACCEPT
👉 But actual traffic = ❌ dropped

---

# 🧠 Important Insight

👉 Flow Logs show:

* Network-level decision

👉 NOT:

* Application-level or service-level behavior
