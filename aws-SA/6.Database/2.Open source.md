  * OpenSearch (engine)
  * OpenSearch Dashboards (visualization)

📌 Used for **fast search, indexing, and analytics**

# 🔥 Important Architecture Pattern

---

## 🔄 DynamoDB + OpenSearch

👉 Problem:

* Amazon DynamoDB
  ❌ No advanced search

👉 Solution:

1. DynamoDB changes → Streams
2. Trigger AWS Lambda
3. Push data → OpenSearch
4. Search via OpenSearch
5. Fetch full data from DynamoDB

---

## 🧠 Why?

* DynamoDB → fast key-value
* OpenSearch → powerful search

👉 Together = complete system

---

# 🔹 Log Analytics Pattern

---

## Option 1 (Real-time)

* Amazon CloudWatch Logs
  → Lambda
  → OpenSearch

---

## Option 2 (Near real-time)

* CloudWatch Logs
  → Amazon Kinesis Data Firehose
  → OpenSearch

👉 Firehose adds batching

# 🔥 OpenSearch vs DynamoDB vs RDS

| Feature    | OpenSearch       | DynamoDB    | RDS        |
| ---------- | ---------------- | ----------- | ---------- |
| Purpose    | Search           | Key-value   | Relational |
| Query type | Full-text        | Key-based   | SQL        |
| Speed      | Very fast search | Fast lookup | Moderate   |

---

# 🧠 Memory Trick

* **OpenSearch = Search engine**
* **DynamoDB = Fast storage**
* **Lambda = Connector**

---

# ⚠️ Exam traps

* Need **full-text search** → OpenSearch
* Need **primary database** → ❌ Not OpenSearch
* Need **search on DynamoDB data** → OpenSearch + Streams
* Need **log analytics** → OpenSearch
