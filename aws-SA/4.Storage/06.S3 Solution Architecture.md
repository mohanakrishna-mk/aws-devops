# 🔹 1. Ways to Serve Static Content

## 🧱 Option 1 — EC2 (basic, not scalable)

* Public EC2 instance
* Data stored on:

  * EBS OR instance store

❌ Problems:

* Not scalable
* Not highly available
* Single point of failure

👉 Only for **testing / small apps** 

---

## 🌍 Option 2 — CloudFront + EC2

👉 Add CDN using Amazon CloudFront

* Global caching
* Faster delivery

⚠️ Still problem:

* If EC2 fails → backend fails

👉 Better, but not best 

---

## ⚙️ Option 3 — CloudFront + ALB + EC2 + EFS

👉 Full scalable architecture:

* CloudFront → CDN
* ALB → load balancing
* EC2 → app servers
* EFS → shared storage

👉 Use Amazon EFS when:

* Multiple EC2 need same files

⚠️ Downsides:

* Complex
* Expensive

---

## 🚀 Option 4 — CloudFront + S3 (BEST PRACTICE)

👉 Most recommended:

* Amazon S3 → stores static content
* CloudFront → global caching

✅ Benefits:

* Highly scalable
* Highly available
* Low cost
* Global performance

👉 This is the **default modern architecture** 

---

# 🔹 2. S3 + Search Problem (Important)

👉 Problem:

* S3 has **NO search/indexing**

👉 Solution:

Use:

* AWS Lambda
* Amazon DynamoDB

---

## 🔄 Flow

1. Upload file → S3
2. S3 event triggers Lambda
3. Lambda:

   * Reads file (or metadata)
   * Stores index in DynamoDB

👉 Now you can:

* Search by date
* Filter by user
* Query metadata

📌 Very common architecture in exams 

---

# 🔹 3. Static vs Dynamic Content (CRITICAL CONCEPT)

## ⚡ Dynamic Content

👉 Goes to:

* API Gateway / ALB / EC2

👉 Features:

* Frequently changing
* Not cached much

---

## 📦 Static Content

👉 Goes to:

* S3 + CloudFront

👉 Examples:

* Images
* Videos
* HTML
* JS bundles

---

## 🔥 Best Practice

👉 **Separate static and dynamic content**

📌 Why:

* Better performance
* Lower cost
* Easier scaling



---

# 🔹 4. Direct S3 Access (Pre-Signed URL)

👉 Instead of proxying via backend:

* Generate **pre-signed URL**
* Client uploads/downloads directly from S3

✅ Benefits:

* Faster
* Cheaper
* Less backend load

---

# 🔹 5. How Everything Connects (Big Picture)

👉 Typical modern architecture:

* DNS → Route 53
* Static → CloudFront → S3
* Dynamic → API Gateway / ALB → EC2 / Lambda
* Storage → DynamoDB
* Events → Lambda

---

# 🧠 Memory Shortcut

* **S3 = static files**
* **CloudFront = speed**
* **Lambda = automation**
* **DynamoDB = search/index**

---

# 🔥 Exam Traps to Remember

* Need **shared storage across EC2** → EFS
* Need **global static delivery** → S3 + CloudFront
* Need **search in S3** → DynamoDB + Lambda
* Need **secure direct upload** → Pre-signed URL
