Here’s a **clear, exam-focused breakdown of AWS DataSync** — this one shows up a lot in scenarios.

---

# 🔹 What is AWS DataSync?

👉 AWS DataSync

* Service to **move & synchronize large amounts of data**
* Works:

  * On-prem → AWS
  * AWS → On-prem
  * AWS → AWS

📌 Key idea:
👉 **Fast, scheduled, metadata-preserving data transfer**

---

# 🔹 What can it connect to?

## 📥 Source / Destination

* NFS (Linux file systems)
* SMB (Windows shares)
* HDFS (big data)
* Other cloud storage

---

## 📦 AWS Targets

* Amazon S3
* Amazon EFS
* Amazon FSx

📌 Supports all storage classes (even Glacier) 

---

# 🔹 How it works (Important)

## 🏠 On-prem → AWS

1. Install **DataSync Agent** (VM)
2. Connect to:

   * NFS / SMB server
3. Agent sends data → DataSync service
4. Data goes → S3 / EFS / FSx

---

## ☁️ AWS → AWS

* No agent required
* Direct service-to-service sync

---

# 🔹 Key Features (Exam Gold)

## ⚡ High Performance

* Up to **10 Gbps per task**

---

## 🕒 Scheduled (IMPORTANT)

* Not real-time
* Runs:

  * Hourly
  * Daily
  * Weekly

👉 There is always **sync lag**

---

## 🔐 Metadata Preservation (VERY IMPORTANT)

* Keeps:

  * File permissions
  * Ownership
  * Timestamps

📌 Supports:

* POSIX (Linux)
* SMB permissions

👉 This is a **big exam differentiator**

---

## 🎛 Bandwidth Control

* Limit network usage if needed

---

# 🔹 Special Case: No Network?

👉 Use:

* AWS Snowcone

* Comes with **DataSync agent pre-installed**

* Move data physically → AWS

📌 Used when:

* No bandwidth
* Remote locations

---

# 🔹 Common Architectures

## 🔄 1. On-prem → AWS migration

* File server → S3 / EFS / FSx

---

## 🔁 2. AWS → AWS sync

* S3 → EFS
* FSx → S3

👉 Used for:

* Backup
* Data pipelines

---

## 🔄 3. Hybrid sync

* AWS ↔ On-prem

---

# 🔥 DataSync vs Other Services (Exam Trap)

| Feature       | DataSync        | Snowball        |
| ------------- | --------------- | --------------- |
| Transfer type | Network         | Physical device |
| Speed         | Fast            | Offline         |
| Metadata      | ✅ Preserved     | ❌ Limited       |
| Use case      | Continuous sync | Bulk migration  |

---

# 🧠 Memory Trick

👉 **DataSync = Smart copy (with permissions)**
👉 **Snowball = Truck delivery**

---

# ⚡ Exam shortcuts

* “Preserve file permissions?” → **DataSync**
* “Sync NFS/SMB to AWS?” → **DataSync**
* “Regular sync schedule?” → **DataSync**
* “No network available?” → **Snowcone + DataSync**
* “Continuous real-time replication?” → ❌ Not DataSync

---

# 🚀 Real-world example

👉 Company migrating file server:

* On-prem SMB → FSx Windows

Solution:

* Install DataSync agent
* Schedule sync
* Cutover after completion
