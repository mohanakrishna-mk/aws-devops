---

# 🔹 What is S3 Analytics?

👉 Also called **Storage Class Analysis**

👉 Feature in Amazon S3 that helps you:

* Understand **how frequently objects are accessed**
* Decide **when to move data to cheaper storage classes**

---

# 🔹 What it actually does

* Analyzes access patterns of objects in a bucket
* Generates **daily reports**
* Shows:

  * Object age
  * Access frequency
  * Storage class usage

📌 Helps answer:
👉 “Should I move this data to Standard-IA?”

---

# 🔴 Important Limitations (Exam traps)

* ✅ Works for:

  * **Standard**
  * **Standard-IA**

* ❌ Does NOT support:

  * One Zone-IA
  * Glacier tiers

---

# 🔹 How it works (simple flow)

1. Enable Storage Class Analysis on a bucket
2. Wait **24–48 hours**
3. Get **CSV reports** with insights
4. Use data to:

   * Create Lifecycle Rules
   * Optimize storage cost

---

# 🔹 Visualization

* Can integrate with Amazon QuickSight
* Build dashboards for:

  * Access trends
  * Data aging
  * Storage optimization

---

# 🔹 Why it matters

👉 It’s a **decision tool**, not automation

* It does NOT move data automatically
* It only gives **recommendations**

👉 You still need:

* Lifecycle Policies to actually move objects

---

# 🔥 Real-world example

* You have:

  * 1 TB of images in S3 Standard
* Analytics shows:

  * 80% not accessed after 30 days

👉 Action:

* Create lifecycle rule → move to Standard-IA after 30 days

---

# 🧠 Simple memory trick

👉 **Analytics = Suggest**
👉 **Lifecycle = Execute**
