Here’s a **simple, exam-focused way to understand AWS storage pricing order** — this is exactly the kind of trick question examiners love.

---

# 🔥 Big Picture: Cost Ranking (Most → Least Expensive)

👉 Across Amazon EBS, Amazon EFS, and Amazon S3

## 💸 From MOST expensive → CHEAPEST

1. **EBS io1 / io2** (highest performance, provisioned IOPS)
2. **EFS Standard / EFS One-Zone**
3. **EBS gp2 / gp3**
4. **EBS st1 (HDD, throughput)**
5. **EFS IA (Standard IA)**
6. **EBS sc1 (cold HDD)**
7. **EFS One-Zone IA (cheapest EFS)**
8. **S3 Standard IA / Glacier / Deep Archive (cheapest overall)**

📌 Summary:
👉 **Performance ↑ → Cost ↑**
👉 **Cold storage ↓ → Cost ↓**

---

# 🔹 Why this order makes sense

## ⚡ High performance = High cost

* EBS io1/io2 → databases, critical apps
* You pay for **low latency + high IOPS**

---

## 🌐 Distributed systems = expensive

* EFS → multi-AZ, shared file system
* More infrastructure → more cost

---

## 💾 HDD = cheaper

* st1 / sc1 → slower, sequential workloads

---

## 🧊 Cold storage = cheapest

* S3 IA / Glacier → rarely accessed
* Trade-off = retrieval cost + latency

---

# 🔴 Important Exam Insight

👉 You **CANNOT compare blindly**
Because:

* EBS = block storage
* EFS = file system
* S3 = object storage

📌 First choose **correct service**, then cheapest option

👉 **“IOPS → File System → General → HDD → Archive”**

# ⚡ Exam shortcuts

* Need **fast DB disk** → EBS io1/io2 (expensive)
* Need **shared storage** → EFS (expensive)
* Need **cheap storage** → S3 IA / Glacier
* Need **lowest cost overall** → S3 Glacier Deep Archive

---

# 🚀 Real-world thinking

👉 If question says:

* “Cheapest storage for rarely accessed data”
  → **S3 Glacier / Deep Archive**

* “Cheapest EBS option”
  → **sc1**

* “Cheapest EFS option”
  → **One-Zone IA**

---

# ⚠️ Common mistake

❌ Choosing cheapest blindly
✅ Always check:

* Access pattern
* Performance need
* Storage type
