# 🔹 1. What is Amazon S3?

👉 **S3 = Object Storage (serverless)**

* Unlimited storage
* Pay-as-you-go
* Used for:

  * Static websites
  * Images, videos, backups

📌 Key idea:
👉 Data stored as **objects (key + value)**
👉 Not a file system (cannot mount like EFS) 

---

# 🔴 Important Limitations (Exam Traps)

❌ Not a POSIX file system → use EFS instead
❌ No file locking
❌ No native search/query
❌ Not good for:

* Dynamic apps
* Many small files
* Low-latency shared access 

---

# 🔹 2. S3 Storage Classes

## 🧊 Main tiers

| Class                | Use                      |
| -------------------- | ------------------------ |
| Standard             | Frequently accessed      |
| Intelligent-Tiering  | Auto move based on usage |
| Standard-IA          | Infrequent access        |
| One Zone-IA          | Cheap, single AZ         |
| Glacier Instant      | Fast archive             |
| Glacier Flexible     | Slow archive             |
| Glacier Deep Archive | Cheapest, very slow      |

📌 You can move objects using **Lifecycle policies** 

---

# 🔹 3. Replication (CRR vs SRR)

| Type | Meaning                  |
| ---- | ------------------------ |
| CRR  | Cross-region replication |
| SRR  | Same-region replication  |

📌 Requirements:

* Versioning must be enabled

## Uses:

* Disaster recovery
* Reduce latency
* Compliance

---

## ⚡ S3 Replication Time Control (RTC)

* 99.99% objects replicated within **15 minutes**
* Alerts via CloudWatch if delayed 

---

# 🔹 4. S3 Event Notifications

👉 React to events like:

* Object created
* Object deleted

## Can trigger:

* Lambda
* SQS
* SNS

---

## 🚀 Advanced: EventBridge integration

* Better filtering (metadata, size, etc.)
* Multiple targets
* Replay events
* More reliability

📌 Much more powerful than basic notifications 

---

# 🔹 5. Performance Basics

👉 Auto scales by default

* ~3,500 writes/sec per prefix
* ~5,500 reads/sec per prefix

📌 Use **multiple prefixes** to scale massively 

---

# 🔹 6. Upload Optimization

## ✅ Multi-Part Upload

* Required > 5GB
* Recommended > 100MB

👉 Benefits:

* Parallel upload
* Retry only failed parts

---

## ✅ Transfer Acceleration

* Upload via nearest edge location
* Uses AWS backbone network

👉 Faster global uploads

---

# 🔹 7. Download Optimization

## ✅ Byte-Range Fetches

* Download file in parts
* Parallel downloads
* Retry only failed chunks

👉 Also useful for:

* Reading file headers only

---

# 🔹 8. Lifecycle Cleanup

* Delete incomplete multipart uploads
* Save storage cost

---

# 🔥 9. S3 vs EBS vs EFS (Important)

| Feature        | S3             | EBS    | EFS          |
| -------------- | -------------- | ------ | ------------ |
| Type           | Object         | Block  | File         |
| Mountable      | ❌ No           | ✅ Yes  | ✅ Yes        |
| Multi-instance | ✅ Yes          | ❌      | ✅ Yes        |
| Scaling        | Unlimited      | Manual | Auto         |
| Use case       | Static, backup | DB, OS | Shared files |

---

# 🧠 Simple Memory Trick

* **S3 → Store files (objects)**
* **EBS → Disk for 1 server**
* **EFS → Shared network drive**

---

# 🚀 Key exam shortcuts

* Need shared file system → **EFS**
* Need persistent disk → **EBS**
* Need scalable storage → **S3**
* Need archive → **Glacier**
* Need event trigger → **S3 + Lambda**
