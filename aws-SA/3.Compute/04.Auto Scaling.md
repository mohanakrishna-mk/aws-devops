# 🧠 What is HPC (High Performance Computing)?

👉 Run **massive computations in parallel** using many machines.

Why AWS is perfect:

* Spin up **1000s of instances quickly**
* Pay only while running
* Destroy everything after job completes 

👉 Think:

> “Throw huge compute at a problem → finish fast → shut down”

---

# 🚀 Common HPC Use Cases

* Machine learning / deep learning
* Weather simulation
* Genomics
* Financial risk modeling 

---

# 🧩 HPC Architecture (Simple View)

```
Data → Transfer → Compute → Network → Storage → Orchestration
```

---

# 📦 1) Data Transfer (Getting data into AWS)

* AWS Direct Connect
  → Continuous high-speed transfer

* AWS Snowball
  → Petabytes (offline transfer)

* AWS DataSync
  → Automated sync (on-prem → AWS)

👉 Exam trick:

* **TB–PB one-time → Snowball**
* **Continuous → Direct Connect**

---

# ⚙️ 2) Compute Layer (Core of HPC)

* Amazon EC2

  * CPU optimized → compute-heavy
  * GPU → ML workloads

* Spot Instances → **massive cost savings**

👉 HPC = **many EC2 instances running together**

---

# ⚡ 3) Networking (MOST IMPORTANT for HPC)

## 🔹 Cluster Placement Group

* Same AZ, same rack
* Ultra low latency (~10 Gbps) 

👉 Required for:

* Distributed computing (nodes talking to each other)

---

## 🔹 Enhanced Networking

### ENA (Elastic Network Adapter)

* Up to **100 Gbps**
* Lower latency, higher throughput 

---

## 🔹 EFA (Elastic Fabric Adapter)

🔥 This is the **HPC-specific upgrade**

* Works with Linux only
* Uses MPI (Message Passing Interface)
* Bypasses OS → ultra low latency 

👉 Exam key:

* ENA = general performance
* EFA = **HPC distributed workloads**

---

# 💾 4) Storage Options

| Storage               | Use                   |
| --------------------- | --------------------- |
| Amazon EBS            | High-performance disk |
| Instance Store        | Ultra fast, temporary |
| Amazon S3             | Large datasets        |
| Amazon EFS            | Shared file system    |
| Amazon FSx for Lustre | ⭐ HPC optimized       |

👉 Key:

* FSx for Lustre = **best for HPC (millions of IOPS)** 

---

# 🔄 5) Orchestration (Running Jobs)

* AWS Batch
  → Runs jobs across multiple EC2

* AWS ParallelCluster
  → Full HPC cluster automation

---
