# 🧠 What is AWS Global Accelerator?

AWS Global Accelerator

👉 It improves performance by using **AWS global private network instead of the public internet**

---

# ⚡ Core Idea

```text id="8q2lqj"
User → Anycast IP → Nearest AWS Edge → AWS private network → Your app
```

## 🔹 Step-by-step:

1. User hits **Anycast IP** (same IP globally)
2. Routed to nearest AWS Edge location
3. Traffic travels via **AWS internal network**
4. Reaches your backend (ALB / EC2 / NLB)

---

# 🧠 Why This Matters

👉 Public internet = unpredictable
👉 AWS network = optimized, low latency

---

# 🔥 Key Features

---

## 🌐 Anycast IP (Important)

* Same IP worldwide
* No DNS changes
* Stable endpoints

---

## ⚡ Performance

* Lowest latency routing
* Reduced packet loss
* Consistent experience

---

## 🔁 Fast Failover

* Health checks built-in
* Failover in **< 1 minute**

# ⚖️ Global Accelerator vs CloudFront

This is the **most common exam confusion**.

---

## 🔵 CloudFront

Amazon CloudFront

👉 Purpose:

* **Cache content at edge**

---

## 🟢 Global Accelerator

👉 Purpose:

* **Speed up connection to backend**

---

## ⚔️ Key Differences

| Feature  | CloudFront             | Global Accelerator    |
| -------- | ---------------------- | --------------------- |
| Type     | CDN                    | Network accelerator   |
| Caching  | ✅ Yes                  | ❌ No                  |
| Protocol | HTTP/HTTPS             | TCP/UDP               |
| Use case | Static/dynamic content | Full app acceleration |
| IP       | Changes (DNS)          | Static Anycast IP     |

---

# 🎯 When to Use Global Accelerator

✅ Use it when:

* Need **static IP for app**

* Need **low latency globally**

* Non-HTTP traffic:

  * Gaming (UDP)
  * IoT (MQTT)
  * VoIP

* Need **fast failover between regions**

---

# ❌ When NOT to Use

* Static content → use CloudFront
* Basic routing → use Route 53

---

# ⚡ Real Example

👉 Scenario:

Users worldwide accessing app in India

Without GA:

With GA:


# 🧠 Decision Rule (Exam Gold)

```text id="w8x4rl"
Static content → CloudFront

Performance + static IP → Global Accelerator
```
