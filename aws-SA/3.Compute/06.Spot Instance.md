# ⚡ What are Spot Instances?

Amazon EC2

👉 Unused AWS capacity sold at **up to 90% discount** 

---

## 🧠 How it works (core idea)

* You set a **max price**
* AWS gives instance if:

  ```
  current price ≤ your max price
  ```
* If price goes above → instance is **terminated**

⏱️ You get:
👉 **2-minute warning before termination** 

---

# 🚀 When to Use Spot (VERY IMPORTANT)

✅ Good for:

* Batch jobs
* Data processing
* CI/CD builds
* Big data / HPC

❌ NOT for:

* Databases
* Critical production apps
* Stateful systems

👉 Because: instances can disappear anytime 

---

# 💡 Mental Model

```text
Spot = cheap but unreliable
On-demand = expensive but stable
Reserved = cheap + stable (long term)
```

# ⚙️ Spot Fleet (Advanced Concept)

👉 Instead of **1 instance**, you request a **fleet**

---

## 🧠 What Spot Fleet does

* Chooses:

  * instance types
  * AZs
* Tries to meet:

  * your capacity
  * your budget

👉 Automatically picks best option for you 

---

# 🎯 Spot Fleet Strategies (Exam Gold)

## 1️⃣ Lowest Price

* Chooses cheapest pool

👉 Best for:

* short jobs
* max savings

---

## 2️⃣ Diversified

* Spreads across multiple pools

👉 Best for:

* stability
* long jobs

---

## 3️⃣ Capacity Optimized

* Picks pool with most available capacity

👉 Best for:

* fewer interruptions

---

## 4️⃣ Price-Capacity Optimized ⭐ (Most important)

* Balance between cost + availability

👉 Best default choice

---

# ⚖️ Spot vs Spot Fleet

| Feature           | Spot Instance | Spot Fleet |
| ----------------- | ------------- | ---------- |
| Control           | Manual        | Automated  |
| Instance types    | One           | Multiple   |
| Availability      | Lower         | Higher     |
| Cost optimization | Medium        | High       |

---

# 🔥 Real DevOps Usage

In real systems, you DON’T use Spot alone.

👉 You combine:

* Auto Scaling Group
* Mix of:

  * On-demand (base capacity)
  * Spot (extra capacity)

---

Example:

```text
Base: 2 On-Demand instances
Scale: 8 Spot instances
```

👉 Result:

* Cheap
* Still reliable
