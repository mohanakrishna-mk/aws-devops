This is a **very dense but high-scoring topic**—API Gateway questions are often about **limits + architecture choices**, not just features. Let’s break it into what actually matters.

---

# 🧠 What is API Gateway (real meaning)

Amazon API Gateway

👉 It’s not just “expose API”
👉 It’s a **control layer in front of your backend**

```text id="p9q3fr"
Client → API Gateway → Backend (Lambda / HTTP / AWS Service)
```

---

# 🔥 Why NOT call backend directly?

Because API Gateway gives:

* Authentication
* Rate limiting
* Versioning
* Caching
* Monitoring

👉 It’s like a **smart front door**

---

# ⚡ 1) Hard Limits (EXAM CRITICAL)

👉 These kill wrong architectures:

| Limit              | Impact                    |
| ------------------ | ------------------------- |
| **29 sec timeout** | Backend must respond fast |
| **10 MB payload**  | No large uploads          |

---

## ❗ Key Insight

```text id="9l0y3r"
Lambda can run 15 min
BUT API Gateway waits only 29 sec
```

👉 So long Lambda ≠ usable via API Gateway



---

# 🚨 3) File Upload Architecture (Classic Trap)

## ❌ Bad Design

```text id="2z2q6h"
Client → API Gateway → S3
```

👉 Fails because:

* 10 MB limit

---

## ✅ Correct Design

```text id="1n9q2r"
Client → API Gateway → Lambda → Pre-signed URL → S3
```

👉 Then:

```text id="n1ps9r"
Client → S3 directly (no limit)
```

---

👉 This is **very common exam question**

# ⚡ 5) Caching (Performance + Cost)

👉 API Gateway can cache responses

---

## Flow:

```text id="7o5bci"
Request → Cache hit → return
        → Cache miss → backend → store
```

---

## Important:

* Default TTL = 300 sec
* Per stage
* Can override per method

---

👉 Benefit:

* Reduce backend load

---

# 🚨 6) Error Codes (VERY IMPORTANT)

## 🔹 4XX → Client issue

* 400 → bad request
* 403 → access denied
* 429 → throttling

---

## 🔹 5XX → Backend issue

* 502 → bad Lambda response
* 503 → backend down
* 504 → timeout (>29 sec)

---

👉 Key:

```text id="ktz2yy"
504 = timeout problem
```

---

---

→ ❌ Direct API Gateway

→ ❌ API Gateway direct

---

## 👉 “Global users”

→ ✅ Edge-optimized

---

## 👉 “Internal API only”

→ ✅ Private API

---

## 👉 “Reduce backend calls”

→ ✅ Enable caching
