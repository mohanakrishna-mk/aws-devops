# 🧠 What is ECR?

Amazon Elastic Container Registry

👉 ECR = **Docker image storage for AWS**

Think:

```text
GitHub → code
ECR → container images
```

---

# 📦 What does ECR store?

* Docker images (your app packaged)
* Multiple versions of same app
* Tagged images (v1, v2, latest)

---

# 🔄 Basic Flow (Very Important)

```text
Developer → Build Image → Push to ECR → ECS pulls → Runs container
```

---

# ⚙️ Integration with ECS

Amazon Elastic Container Service

👉 ECS does NOT store images
👉 It **pulls images from ECR**

How?

* EC2 instance role (or task role)
* IAM permission: `ecr:GetDownloadUrlForLayer`, etc.

👉 If image pull fails:
→ **99% IAM issue**

---

# 🌍 Types of Repositories

## 🔹 Private

* Your own images
* Secure via IAM

## 🔹 Public

* Shared globally
* Example: `gallery.ecr.aws`

# 🔁 Cross-Region Replication (Exam Favorite)

👉 Build once → use everywhere

Flow:

```text
CodeBuild → ECR (Region A) → Replicated → ECR (Region B)
```

---

## ✅ Why this matters

* No need to rebuild images
* Faster deployment in other regions
* Multi-region architecture

* Checks common vulnerabilities (CVEs)
* Built into ECR

---

## 🔹 Enhanced Scanning

Amazon Inspector

* Deep scan (OS + language)

---

## ⚡ Scan Modes

* Manual scan

---

## 🔔 Results

Git → CodeBuild → Build Docker Image → Push to ECR

# ⚠️ Common Mistakes (Important)

❗ ECS cannot pull image
→ IAM role missing permissions

❗ Using wrong region
→ Image not found

❗ No replication
→ Multi-region fails
