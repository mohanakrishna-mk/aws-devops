# 📜 AWS CloudWatch Logs — Deep Dive

## 📥 1. How Logs Get Into CloudWatch

There are **3 main ways** to send logs:

### 🧰 Agents (most common for servers)

* CloudWatch Logs Agent
* CloudWatch **Unified Agent** (modern, recommended)

👉 Installed on:

* EC2
* On-prem servers
* Any VM

---

### 🔌 AWS Service Integrations (automatic)

Logs are directly pushed by services like:

* AWS Lambda → function logs
* Amazon ECS → container logs
* AWS Elastic Beanstalk → app logs
* Amazon API Gateway → access logs
* AWS CloudTrail → API activity
* Amazon Route 53 → DNS query logs

---

### 🧑‍💻 SDK

* Apps can push logs programmatically

---

## 🧱 2. Log Structure (Very Important)

CloudWatch Logs is structured like this:

```
Log Group (application)
   ├── Log Stream (instance/container)
   │      ├── Log Events (actual logs)
```

### Key ideas:

* **Log Group** → app/service level
* **Log Stream** → per instance/container
* **Log Events** → actual log lines

---

## ⏳ 3. Retention & Security

* Logs can:

  * Never expire
  * Expire after X days

* Encryption:

  * Supported via **KMS**

👉 Real-world:

* Short retention → save cost
* Long-term → export to S3

---

## 🔍 4. Metric Filters (Exam Favorite)

Turn logs → metrics

### Example:

* Count `"ERROR"` occurrences
* Track specific IP access
* Detect security patterns

👉 Then:

* Create **CloudWatch Alarm**
* Trigger automation

---

## 📊 5. Logs Insights (Powerful Tool)

Query logs using SQL-like syntax

### Example use cases:

* Errors per 5 minutes
* Find failed requests
* Debug Lambda issues

👉 Can also:

* Save queries
* Add to dashboards

---

## 📤 6. Exporting Logs (Two Approaches)

---

### ❌ 1. S3 Export (Batch — NOT real-time)

* Uses API: `CreateExportTask`
* Delay: up to **12 hours**
* Must use encryption (SSE-S3 / SSE-KMS)

👉 Use when:

* Archival
* Compliance
* Backup

---

### ⚡ 2. Subscription Filters (Real-Time / Near Real-Time)

This is **VERY IMPORTANT for architecture**

Logs → streamed immediately to:

---

### 🔹 Option A: Lambda (Real-time)

* Instant processing
* Full flexibility

👉 Example:

* Detect error → trigger fix

---

### 🔹 Option B: Kinesis Data Firehose (Near real-time)

* ~1 minute delay
* Cheaper & scalable

👉 Sends to:

* S3
* OpenSearch
* Redshift

---

### 🔹 Option C: Kinesis Data Streams

* Full control pipeline
* Custom processing

---

## 🧠 Decision Rule (Important)

| Need                     | Use          |
| ------------------------ | ------------ |
| Real-time processing     | Lambda       |
| Near real-time + cheaper | Firehose     |
| Full custom pipeline     | Data Streams |

---

## 🌍 7. Multi-Account Log Aggregation (Advanced)

### Architecture:

* Multiple accounts + regions
* Each sends logs → central account

Flow:

```
CloudWatch Logs
   → Subscription Filter
      → Kinesis Data Stream
         → Firehose
            → S3 (central logging account)
```

👉 Used for:

* Security monitoring
* Centralized logging
* Compliance

---

## ⚙️ 8. Installing CloudWatch Agent (Important for DevOps)

Using **AWS Systems Manager**:

### Methods:

* Run Command → install agent
* State Manager → enforce installation
* Config via Parameter Store

👉 Best practice:

* Automate everything via SSM

## 🧠 Real-World Mental Model

Think:

```
Logs → Filter → Metric → Alarm → Action
```

Example:

```
App logs error
 → Metric Filter counts errors
 → Alarm triggers
 → Lambda fixes issue
```
