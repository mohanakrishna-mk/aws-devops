# ⚡ Amazon EventBridge — Deep Dive

## 🧠 What It Is (Simple)

**Amazon EventBridge** is an **event routing system**:

```text
Something happens → Event → EventBridge → Action
```

👉 It connects AWS services together **without tight coupling**

---

## 🔁 1. Two Main Ways It Works

### ⏰ 1. Scheduled Events (Cron Jobs)

* Run tasks on schedule

👉 Example:

* Every hour → trigger **AWS Lambda**
* Daily backup script
* Weekly report generation

---

### ⚡ 2. Event-Based Triggers (Most Important)

Triggered when something happens in AWS:

Examples:

* EC2 instance starts/stops
* File uploaded to S3
* API call via CloudTrail
* Root login detected

👉 Example:

```text
IAM Root Login → EventBridge → SNS → Email Alert
```

---

## 🧩 2. Event Flow Architecture

```text
Event Source → EventBridge Rule → Target
```

### Sources:

* AWS services (EC2, S3, CodeBuild, etc.)
* Custom apps
* SaaS tools

---

### Targets:

EventBridge can trigger:

* AWS Lambda
* Amazon SNS
* Amazon SQS
* AWS Step Functions
* Amazon ECS
* AWS CodePipeline

👉 This is why EventBridge is called:

> **“Glue of AWS services”**

---

## 🔍 3. Event Filtering (Very Important)

You don’t trigger on every event — you filter them.

👉 Example:

* Only events from a specific S3 bucket
* Only failed builds
* Only root user login

---

## 🧭 4. Event Buses (3 Types)

---

### 1️⃣ Default Event Bus

* Built-in
* Receives AWS service events

---

### 2️⃣ Partner Event Bus

* From SaaS tools

Examples:

* Datadog
* Zendesk
* Auth0

👉 External apps → EventBridge

---

### 3️⃣ Custom Event Bus

* Your own applications send events

👉 Example:

```text
Your Node.js app → Custom Event → EventBridge → Lambda
```

---

## 📦 5. Event Format

Events are sent as **JSON documents**

Contain:

* Source
* Time
* Resource ID
* Metadata

👉 This is important for filtering + automation

---

## 📚 6. Schema Registry (Advanced)

EventBridge can:

* Detect event structure automatically
* Store schemas
* Generate code

👉 Benefit:

* Your app knows event structure in advance
* Faster development

---

## 🔁 7. Archive & Replay (🔥 Powerful Feature)

### Archive:

* Store past events

### Replay:

* Re-run old events

👉 Real use case:

```text
Bug in Lambda → Fix code → Replay old events
```

⚡ Super useful for:

* Debugging
* Recovery
* Testing

---

## 🔐 8. Resource-Based Policies (Cross Account)

Allows:

* Other AWS accounts → send events to your bus

### Example:

```text
Multiple accounts → Central Event Bus
```

👉 Used in:

* Organizations
* Centralized logging
* Security monitoring

---

## 🧠 Real-World Example

### 🔐 Security Alert System

```text
CloudTrail detects root login
 → EventBridge rule
 → SNS
 → Email alert
```

---

### 🚀 Auto-Healing System

```text
EC2 failure event
 → EventBridge
 → Lambda
 → Restart instance
```

---

### 📦 CI/CD Automation

```text
Code commit
 → EventBridge
 → CodePipeline trigger
```

---

## ⚡ Exam Cheat Sheet

* EventBridge = **event router**
* Supports:

  * Scheduled events (cron)
  * Event-driven triggers
* Event buses:

  * Default (AWS)
  * Partner (SaaS)
  * Custom (your apps)
* Can trigger MANY AWS services
* Supports filtering (VERY important)
* Archive + Replay = debugging
* Schema Registry = structure awareness
* Cross-account via resource policies
