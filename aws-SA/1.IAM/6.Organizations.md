# 🏢 What is AWS Organizations?

👉 It lets you:

> **Manage multiple AWS accounts centrally**

Instead of one big account → you split into multiple accounts and control them from one place 

---

# 🧱 Structure of AWS Organizations

```
Root
 ├── Management Account (admin)
 ├── OU (Dev)
 │    ├── Account 1
 │    ├── Account 2
 ├── OU (Prod)
 │    ├── Account 3
 │    ├── Account 4
```

---

## 🔑 Key Components

### 1️⃣ Management Account

* Main controlling account
* Handles billing + governance

---

### 2️⃣ Member Accounts

* Normal AWS accounts
* Used for workloads

---

### 3️⃣ OU (Organizational Units)

* Group accounts logically:

  * Dev / Test / Prod
  * HR / Finance
  * Projects

---

# 🔐 OrganizationAccountAccessRole (VERY IMPORTANT)

👉 Automatically created when account is added via API

### What it does:

* Exists in **member account**
* Allows **management account to control it**

---

## 🔄 How it works:

1. Management account calls **AssumeRole**
2. Uses `OrganizationAccountAccessRole`
3. Gets **admin access to member account**

✔ Used for automation/admin tasks 

---

## ⚠️ Important Edge Case

* If you **invite an existing account** → role is NOT created
  👉 You must create it manually

---

# 🧠 Why Multiple Accounts? (Real Reasons)

Instead of 1 account:

### ✅ Better isolation

* Prod != Dev

### ✅ Security boundaries

* Blast radius reduction

### ✅ Billing separation

* Cost per team/project

### ✅ Service limits

* Each account has its own limits

---

# 💰 Consolidated Billing

👉 One bill for all accounts

### Benefits:

* Volume discounts (EC2, S3, etc.)
* Centralized payment

✔ All usage combined → more savings 

---

# ⚙️ Feature Modes (Exam Favorite)

## 1️⃣ Consolidated Billing Mode

* Only billing aggregation

---

## 2️⃣ All Features Mode ✅

* Includes billing
* Adds **Service Control Policies (SCPs)**

👉 Required for advanced control

---

## ⚠️ Important:

* Once enabled → ❌ cannot go back

---

# 🔒 Reserved Instances & Savings Plans Sharing

👉 By default:

* Discounts are shared across accounts

### Example:

* Account A buys RI
* Account B uses it

✔ Organization gets max savings

---

## ⚠️ You Can Disable Sharing

* Per account basis
* Useful for strict billing separation

---

# 🔄 Moving Accounts Between Organizations

👉 Simple process:

1. Send invite from new org
2. Accept from member account

✔ No need to manually remove first anymore

# 🧠 Simple Way to Remember

* **Organizations = Multi-account manager**
* **OU = Folder of accounts**
* **Management account = Root admin**
* **SCP = Guardrails (not permissions)**
