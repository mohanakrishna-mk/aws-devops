Here’s a **clear, exam-focused + real-world breakdown of SCP (Service Control Policies)** — this is one of the most misunderstood AWS topics, so let’s simplify it properly 👇

---

# 🔒 What is an SCP?

👉 SCP = **Guardrail for accounts in AWS Organizations**

> It defines **what accounts are allowed or NOT allowed to do**

✔ Applied at:

* Root
* OU
* Account

✔ Affects:

* All users
* All roles (even root user) 

---

# 🚨 Important Rules (Must Remember)

### ❗ 1. SCP does NOT grant permissions

* It only **limits permissions**

👉 IAM policy must allow + SCP must allow

---

### ❗ 2. Explicit DENY = Always wins

* Same as IAM

---

### ❗ 3. Management Account is NOT affected

* SCPs don’t restrict it
* Prevents lockout 

---

### ❗ 4. Service-linked roles are NOT affected

* AWS services still work normally

---

# 🧠 How SCP Evaluation Works (Simple Version)

For an action to work:

✔ IAM policy → must allow
✔ SCP → must allow
✔ No explicit deny anywhere

👉 If any layer denies → ❌ blocked

---

# 🧱 Example Structure

```id="k7r4zz"
Root (FullAWSAccess)
 ├── OU: Sandbox (Deny S3)
 │    ├── Account A (Deny EC2)
 │    ├── Account B
 │    ├── Account C
```

---

## 🔍 Result:

### Account A:

* ❌ No S3 (OU level deny)
* ❌ No EC2 (account level deny)

---

### Account B & C:

* ❌ No S3
* ✅ Everything else

---

✔ Denies **inherit downward** 

---

# ⚙️ Allow vs Deny Strategy

---

## 🔴 Blocklist (Most Common)

```json id="wr7jcs"
Allow: *
Deny: DynamoDB
```

👉 Allow everything except DynamoDB

---

## 🟢 Allowlist (Strict)

```json id="6ogjz6"
Allow: EC2, CloudWatch
```

👉 Only these services allowed

---

# 🎯 Common Use Cases

### 🚫 Restrict services

* Block EMR / DynamoDB

---

### 🌍 Restrict regions

```json id="0wxr99"
Condition: aws:RequestedRegion != allowed regions
```

👉 Prevent usage outside approved regions

---

### 🏷️ Enforce tagging

* Require:

  * `Env`
  * `CostCenter`

👉 Block resource creation without tags

---

# 🧪 Tag Enforcement (Exam Favorite)

### Require BOTH tags:

```json id="jfy1t2"
ForAllValues:StringEquals
```

👉 Must include all tags

---

### Require ANY tag:

```json id="gx8p2h"
ForAnyValue:StringEquals
```

👉 At least one tag

---

# 🚫 Deny if tag missing

```json id="av8yxu"
"Null": {
  "aws:RequestTag/Project": "true"
}
```

👉 Block resource creation without tag

---

# 🏷️ Tag Policies (Different from SCP)

👉 Used for:

* Standardizing tags
* Enforcing allowed values

✔ Example:

* CostCenter = only `100` or `200`

✔ Helps:

* Billing
* Organization-wide consistency 

---

# 🤖 AI Services Opt-Out Policy

👉 Prevent AWS from using your data to improve AI

Example:

* Disable:

  * Rekognition
  * Lex

✔ Important for compliance

---

# 💾 Backup Policies

👉 Define backups centrally

* Frequency
* Region
* Retention

✔ Managed from organization level
✔ Read-only in member accounts

---

# 🧠 Simple Way to Remember

* **IAM = Permissions**
* **SCP = Boundaries (limits)**

👉 Final access = Intersection of both

---

# 🔥 Real DevOps Usage

* Block production account from:

  * Deleting logs
  * Using unsafe regions

* Enforce:

  * Mandatory tags
  * Security standards

* Restrict:

  * Experimental services in prod
