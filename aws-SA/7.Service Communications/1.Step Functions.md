## 🧠 What are Step Functions?

AWS Step Functions is a **serverless workflow orchestration service** used to coordinate multiple AWS services (especially Lambda) into a **state machine**.

👉 Think:
Instead of writing complex glue code → you visually define the flow.

---

## 🔑 Core Idea

* Define workflow as a **state machine (JSON)**
* Each step = a **state**
* Automatically handles:

  * sequencing
  * retries
  * errors
  * parallel execution

📌 From your notes:

* Max execution time = **1 year** (standard workflow) 
* Can include **human approval steps**

---

## ⚙️ What Step Functions Can Do

### 1. Orchestration Features

* Sequential execution
* Parallel execution
* Conditional branching
* Timeouts & waits
* Error handling & retries

---

### 2. Types of Tasks

**Lambda Task**

* Invoke Lambda functions

**Service Task**

* Direct integration (no Lambda needed):

  * DynamoDB
  * SQS / SNS
  * ECS / Fargate
  * Batch
  * Glue, EMR, SageMaker

**Activity Task**

* External workers (EC2 / on-prem)
* ❗ Not serverless

**Wait Task**

* Pause execution (time-based)

## 🚀 How to Trigger Step Functions

* AWS SDK / CLI
* API Gateway (REST call → workflow)

* Lambda
* CodePipeline

---

## 🔄 Standard vs Express Workflows

### 🟢 Standard Workflow

* Max duration: **1 year**
* Execution: **Exactly once**
* Lower throughput
* Full execution history
* Pricing: per **state transition**

👉 Use when:

* Long-running workflows
* Critical processes

---

### ⚡ Express Workflow

* Max duration: **5 minutes**
* Execution: **At least once**
* Very high throughput (100K/sec)
* No full history (use CloudWatch logs)
* Pricing: execution + duration

👉 Use when:

* High-volume events
* Real-time processing

---

## 🔁 Express Modes

### Sync

* Waits for result
* Example: API Gateway → response returned

### Async

* Fire-and-forget
* Example: background processing

---

## ⚠️ Error Handling Pattern

Inside Step Functions:

* Retry logic
* Catch blocks

Outside:

* Use EventBridge → SNS → Email alerts

📌 Common pattern from your notes:

1. Lambda fails
2. Retry configured
3. Still fails
4. EventBridge triggers SNS notification

## 📌 Important Exam Traps

❗ Step Functions ≠ Mechanical Turk

* Use **SWF (Simple Workflow Service)** for Mechanical Turk

❗ Latency exists between Lambda calls

* Because data is passed between states
