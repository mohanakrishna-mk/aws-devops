Here’s a **clean, exam-focused breakdown of advanced Amazon CloudFront features** — these are the ones that show up in tricky questions.

---

# 🔹 1. Geo Restriction (Geo Blocking)

👉 With Amazon CloudFront you can:

* ✅ Allow only specific countries
* ❌ Block specific countries

📌 Uses Geo-IP database to detect user location 

---

## 🔧 Types

* **Allow list** → only selected countries
* **Block list** → deny certain countries

---

## 🎯 Use cases

* Copyright restrictions
* Regional content licensing

---

## 🔥 Bonus (Exam Trick)

CloudFront adds header:
👉 `CloudFront-Viewer-Country`

➡️ Use it in:

* Backend logic
* Lambda@Edge

---

# 🔹 2. Price Classes (Cost Optimization)

👉 Not all edge locations cost the same

📌 Cheapest → US / Canada
📌 Expensive → regions like India 

---

## 💰 Price Classes

| Class | Coverage              | Cost    |
| ----- | --------------------- | ------- |
| All   | Global                | Highest |
| 200   | Most regions          | Medium  |
| 100   | Cheapest regions only | Lowest  |

---

## 🧠 Memory

* **100 → cheapest**
* **All → best performance**

---

# 🔹 3. Signed URL (Secure Access)

👉 Restrict access to content

---

## 🔄 Flow

1. Client requests access
2. App server authenticates user
3. Generates **signed URL**
4. Client uses URL → access CloudFront

---

## 🔐 Features

* Expiration time
* IP restrictions
* Path restrictions

📌 Works for ANY origin (not just S3)

---

## 🔥 Signed URL vs S3 Pre-Signed URL

| Feature | CloudFront          | S3      |
| ------- | ------------------- | ------- |
| Scope   | CDN content         | S3 only |
| Control | Advanced (IP, path) | Basic   |
| Cache   | Yes                 | No      |

---

# 🔹 4. Custom Error Pages

👉 Replace default errors (403, 404, 500)

---

## 🔄 Flow

1. Origin returns error
2. CloudFront intercepts
3. Returns custom page (from S3)

---

## ✅ Benefits

* Better UX
* Branding
* Can cache errors

📌 Stored typically in S3 

---

# 🔥 Key Exam Patterns

* “Restrict by country” → **Geo restriction**
* “Reduce CloudFront cost” → **Price class**
* “Temporary secure access” → **Signed URL**
* “Custom error experience” → **Custom error pages**

---

# 🧠 Memory Shortcut

* **Geo → location control**
* **Price → cost control**
* **Signed URL → access control**
* **Error page → UX control**

---

# 🚀 Real-world example

👉 Streaming app:

* Geo restriction → only licensed countries
* Signed URLs → paid users only
* Price class → reduce CDN cost
* Custom errors → branded pages
