# 🔹 What is CloudFront?

👉 Amazon CloudFront

* **CDN (Content Delivery Network)**
* Caches content at **edge locations globally**
* Improves:

  * Performance
  * Latency
  * Availability

📌 200+ edge locations worldwide 

---

# 🔹 How it works

## 🌍 Basic Flow

1. User requests content
2. Request goes to nearest **edge location**
3. If cached → return instantly
4. If not → fetch from origin → cache

👉 Result:

* Faster access
* Reduced load on backend

---

# 🔹 Key Benefits

## ⚡ Performance

* Cache at edge → low latency

## 🌐 Global availability

* Users served from nearest region

## 🔐 Security

* Built-in:

  * AWS Shield
  * AWS WAF
* HTTPS by default

## 🔁 Scalability

* Handles massive traffic automatically

---

# 🔹 Origins (VERY IMPORTANT)

CloudFront can fetch data from:

---

## 📦 1. S3 (Most common)

* Static content:

  * Images
  * Videos
  * HTML

👉 Use **OAC (Origin Access Control)** to secure S3

---

## 🖥️ 2. Custom Origin (HTTP)

* EC2
* ALB
* API Gateway
* Any public HTTP server

---

## 🔒 3. VPC Origin (Private)

* Private EC2 / ALB inside VPC
* No public exposure

👉 Best for secure architectures

---

## 🎥 4. Media Services

* MediaStore / MediaPackage
* For streaming

---

# 🔥 CloudFront vs S3 Cross-Region Replication

| Feature | CloudFront      | S3 Replication   |
| ------- | --------------- | ---------------- |
| Type    | Cache           | Copy             |
| Scope   | Global          | Specific regions |
| Use     | Static delivery | DR / data sync   |
| Latency | Very low        | Low              |

📌 Completely different use cases 

---

# 🔹 Security Patterns (Exam Favorite)

## 🔐 1. Restrict direct access

👉 Problem:
Users bypass CloudFront → hit origin directly

👉 Solution:

* Use **custom HTTP header**
* Origin only accepts requests with secret header

---

## 🔐 2. Restrict by IP

* Allow only CloudFront IPs in security group

---

## 🔐 3. Private S3 access

* Use OAC
* Block public access

---

# 🔹 High Availability (Origin Groups)

👉 Setup:

* Primary origin
* Secondary origin

## 🔄 Failover flow

1. Try primary
2. If fails → fallback to secondary

📌 Can be cross-region (DR)
