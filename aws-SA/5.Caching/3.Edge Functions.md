# 🔹 What are Edge Functions?

👉 With Amazon CloudFront, you can run code **at the edge (near users)**

👉 Purpose:

* Modify requests/responses
* Authenticate users
* Optimize caching
* Reduce latency

📌 Runs globally, serverless, no infrastructure to manage 

---

# 🔥 Two Types (VERY IMPORTANT)

---

## ⚡ 1. CloudFront Functions

👉 Ultra-fast, lightweight

### Key Features:

* Runs at **Edge Locations**
* Language: JavaScript only
* Execution time: **~1 ms**
* No network calls
* No external APIs

---

### ✅ Use cases:

* Header manipulation
* URL rewrites / redirects
* Cache key normalization
* Simple auth (JWT validation)

---

## 🚀 2. Lambda@Edge

👉 More powerful, flexible

### Key Features:

* Runs at **Regional Edge Cache**
* Languages: Node.js, Python
* Execution time:

  * Viewer: ~5 sec
  * Origin: ~30 sec
* Supports:

  * Network calls
  * AWS SDK
  * Request body access

---

### ✅ Use cases:

* Call external APIs
* Complex authentication
* Dynamic content generation
* Backend integration

---

# 🔥 Key Difference Table

| Feature        | CloudFront Functions | Lambda@Edge    |
| -------------- | -------------------- | -------------- |
| Speed          | Ultra fast           | Slower         |
| Execution      | ~1 ms                | Up to 30 sec   |
| Location       | Edge                 | Regional cache |
| Network access | ❌ No                 | ✅ Yes          |
| Complexity     | Simple               | Complex        |
| Cost           | Cheaper              | More expensive |

---

# 🔹 Where they run (Important)

* **CloudFront Functions** → Viewer request/response
* **Lambda@Edge** → Viewer + Origin request/response

📌 Placement matters for exam questions 

---

# 🔹 When to use which?

## 🧠 Use CloudFront Functions when:

* Need ultra-low latency
* Simple logic
* High scale (millions req/sec)

👉 Example:

* Redirect `/mobile` users
* Add/remove headers

---

## 🧠 Use Lambda@Edge when:

* Need external API calls
* Need request body access
* Need complex logic

👉 Example:

* Authenticate with external system
* Modify response dynamically

---

# 🔥 Real-world examples

---

## 📱 Device-based optimization

* Detect mobile user
* Serve smaller image

👉 Done using CloudFront Functions

---

## 🔐 Authentication

* Validate JWT at edge
* Reject invalid users early

👉 CloudFront Functions (simple)
👉 Lambda@Edge (complex auth)

---

## 🌍 Global API

* CloudFront + Lambda@Edge
* Fetch data from DynamoDB

👉 Fully serverless global backend

---

# ⚠️ Exam traps

* Need **network access** → ❌ CloudFront Functions → ✅ Lambda@Edge
* Need **very low latency** → ✅ CloudFront Functions
* Need **request body access** → ✅ Lambda@Edge
* Need **cheap + fast** → CloudFront Functions

---

# 🧠 Memory Trick

👉 **Functions = Fast & Simple**
👉 **Lambda@Edge = Powerful & Flexible**

---

# ⚡ Final shortcut

* Simple header/URL change → **CloudFront Functions**
* Complex backend logic → **Lambda@Edge**
